Regular Expressions

Regular expressions are patterns used to match character combinations in strings.
Source: Mozilla Foundation

To form a regular expression, you reference a sequence of letters and use special characters ( /|*?+\ etc) to specify their pattern. In ruby, regular expressions are typically delimited with forward slashes (/). The backslash (\) will create an escape sequence and reverse the roles of letters and symbols. Letters preceeded by backslashes (/w) are used to specify patterns, while special characters preceeded by a backslash (\*) are a literal reference to that symbol.

Many programming languages have dedicated libraries that provide regular expresion funcationality. A few, like Ruby, even have regualr expressions built in. In Ruby, you can use the #.match method or the =~ operator to check if a string contains a regular expression pattern. For example:

/ruby/.match('ruby is awesome!')
=> <#MatchData "ruby">
/python/.match('ruby is awesome!')
=> nil
/ruby/ =~'ruby is awesome!'
=> 0
/python/ =~'ruby is awesome!'
=> nil
one more way!
"ruby is awesome!"[/ruby/]
=> "ruby"
"ruby is awesome!"[/python/]
=> nil

String interpolation can be used to to form a regular expression as well. For example:

name = "jeff"
/#{name}/.match("jeff is kewl")
=> <#MatchData "jeff">

Typically, when letters are adjacent to eachother within a regular expression the string must match that entire sequence of characters. \or\ will match store, road, but not broad. However, placing letters inside square prackets ([,]) will create a character class and ruby will only require one of the letters within that class to match. /[or]/ would match broad just fine!

More commonly used characters include:
* - Matches the preceeding character 0 or more times
+ - Matches the preceeding character 1 or more times
? - Matches the preceeding character 0 or 1 time
^ - References the beginning of input
$ - Matches the end of input
. - Matches any character except 'newline'
| - Matches either x or y in the expression x|y
() - Matches whatever is between () and stores the result to be referenced later

We've only scratched the surface here with regular expressions. There are enough letters and symbols available to create nearly an infinite number of regular expression patterns, so fully understanding them requires time and practice. To get started, jump into IRB and use the .match method or the =~ operator. Good luck!
